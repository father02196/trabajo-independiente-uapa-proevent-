# Reglas de Comportamiento del Proyecto

## 1. Mostrar números de línea al proponer cambios
A partir del 17 de Julio de 2026: 
Siempre que se presenten fragmentos de código, planes de solución en Markdown o propuestas de edición al usuario, es OBLIGATORIO incluir explícitamente los números de línea exactos donde se encuentra el código en el archivo destino. Esto ayuda al usuario a localizar rápidamente el bloque si desea hacer el cambio de forma manual.

Ejemplo:
```javascript
// Líneas 60-70 en src/pages/LicitacionesElegidas.js
<input 
  id="search-licitaciones"
  name="search-licitaciones"
  ...
/>
```
